ğŸš€ Running the Full Application

You'll start 3 things in order:

  1. Docker containers (MongoDB + Redis)
  2. Backend server (Express API)
  3. Frontend (HTML in browser)

1 Start Docker Containers

Open a NEW terminal and navigate to project root:

cd webapp # Start MongoDB and Redis docker-compose up -d

The -d means "detached mode" (runs in background)

Verify containers are running:

docker-compose ps

You should see:

NAME SERVICE STATUS PORTS mongodb mongodb running 0.0.0.0:27017->27017/tcp redis redis running 0.0.0.0:6379->6379/tcp

2 Start Backend Server

Open a NEW terminal (keep Docker terminal running):

cd webapp/backend # Start backend in development mode npm run dev

You should see:

[INFO] Server running on port 5000 [INFO] MongoDB connected: localhost [INFO] Redis connected: localhost:6379
ğŸ’¡ Keep this terminal open! Don't close it while using the app.
To stop backend later: Press Ctrl + C

3 Start Frontend

Option A: Simple HTTP Server (Recommended)

Open a NEW terminal (keep backend terminal running):

cd webapp/frontend # Start a simple web server python3 -m http.server 3000 # Or if you have Node.js http-server npx http-server -p 3000

Option B: Open Directly in Browser

  1. Navigate to webapp/frontend folder
  2. Right-click on index.html
  3. Choose "Open With" â†’ Your web browser
âš ï¸ Note: Some features may not work without a proper server due to CORS restrictions. Option A is recommended.

How It Works

1. User Registration & Login

  • User enters credentials in frontend form
  • Frontend sends POST request to /api/v1/auth/register or /login
  • Backend validates data, hashes password
  • Creates user in MongoDB with 50 trial credits
  • Returns JWT token
  • Frontend stores token in LocalStorage
  • Token included in all future requests

2. Creating Presentation

  • User clicks "Create Presentation"
  • Enters title and prompt OR uploads document
  • Frontend sends request to /api/v1/presentations
  • Backend checks user credits
  • Deducts credits from user
  • Sends prompt to Google Gemini AI
  • AI generates slide outline
  • Saves presentation to MongoDB
  • Returns presentation data
  • Frontend displays success

ğŸ› Troubleshooting

Problem: "Cannot connect to MongoDB"

Error: connect ECONNREFUSED 127.0.0.1:27017

Solutions:

  1. Check if Docker is running: docker ps
  2. Start containers: docker-compose up -d
  3. Verify MongoDB: docker-compose ps
  4. Check Docker Desktop - should have green icon

Problem: "CORS Error" in Browser

Access to XMLHttpRequest blocked by CORS policy

Solutions:

  1. Check backend/.env: CORS_ORIGIN=http://localhost:3000
  2. Ensure backend runs on port 5000
  3. Ensure frontend runs on port 3000
  4. Restart backend after changing .env

Problem: "Gemini API Error"

Error: API key not valid

Solutions:

  1. Check API key in backend/.env
  2. Verify at https://makersuite.google.com/app/apikey
  3. Generate new key if needed
  4. Restart backend: Press Ctrl+C, then npm run dev

Problem: "Port 5000 already in use"

Error: listen EADDRINUSE :::5000

Solutions:

Mac/Linux:

lsof -i :5000 kill -9 <PID>

Windows:

netstat -ano | findstr :5000 taskkill /PID <PID> /F

Problem: "Login not working"

Solutions:

  1. Check browser console (F12) for errors
  2. Verify backend: curl http://localhost:5000/health
  3. Check MongoDB is running
  4. Verify API_URL in frontend/app.js
